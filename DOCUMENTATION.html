sf org list metadata --metadata-type ApexClass

ğŸ“– Usage Guide

Use Case 1: Creating a Deployment Package

Scenario: Deploy custom object changes from DEV to UAT

  1. Open Deployment Wizard component
  2. Step 1: Enter package name and select target org
  3. Step 2: Select components to deploy
  4. Step 3: Configure deployment options (test level, rollback)
  5. Step 4: Review and initiate deployment
  6. Monitor progress in Deployment Monitor

Use Case 2: Filtering Audit Trails

Scenario: Find all changes made to custom fields in the last 7 days

  1. Open Audit Trail Viewer
  2. Set date range (last 7 days)
  3. Filter by Metadata Type: "CustomField"
  4. Click Search
  5. Review results in datatable

Use Case 3: Exporting Processing Logs

Scenario: Generate report of failed operations

  1. Open Processing Log Viewer
  2. Filter Status: "Failed"
  3. Click "Export to CSV"
  4. Open file in Excel

handleSearchChange(event) {
    clearTimeout(this.searchTimeout);
    this.searchTimeout = setTimeout(() => {
        this.applyFilters();
    }, 300); // 300ms debounce
}

ğŸ› Troubleshooting

Common Issues

Issue 1: "Cannot read property of undefined"

Cause: Wire adapter returned null before data

Solution:

if (data && data.length > 0) {
    this.records = data;
}

Issue 2: Deployment Fails with "Invalid Type"

Cause: Metadata type not supported

Solution: Check MetadataTypeConfig__mdt configuration

Issue 3: "Too many SOQL queries"

Cause: N+1 query problem

Solution: Add cacheable=true or batch queries

Issue 4: LWC Component Not Loading

Solution: Verify metadata XML exists with correct configuration

Issue 5: Polling Updates Not Working

Solution: Ensure polling interval â‰¥ 10 seconds
